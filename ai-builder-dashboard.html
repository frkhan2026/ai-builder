Cloud-hosted morning brief Scoped High

Pulls from Outlook calendar, Outlook inbox (including Zoom summaries), and Slack via Chrome each weekday morning. Claude extracts action items, estimates time, separates tasks from FYIs, and delivers a styled HTML email to your Outlook inbox before you arrive — matching the layout of your personal brief.

Sub-builds
1 Calendar pull 1–2 hrs
Chrome reads Outlook calendar for today + this week, formats "On your calendar" HTML block. Done when: correct HTML section outputs for any weekday test run.
2 Slack + Outlook scanner 2–3 hrs
Reads Slack + Outlook email (prioritising Zoom summaries). Claude extracts action items with time estimates and FYIs. Done when: ≥3 prioritised items output correctly from a live session.
3 Assembly + delivery 1–2 hrs
Combines all sections into styled HTML email, sends to Outlook inbox. Done when: complete brief arrives in Outlook on a test weekday run.
Starting prompt — Sub-build 1
Build a Python script called work_brief_calendar.py that: 1. Uses Claude in Chrome MCP to open or find an Outlook Web tab at outlook.office.com/calendar 2. Navigates to the weekly calendar view 3. Reads all events from today through end of the current work week (Mon–Fri) 4. Formats them into an HTML <section class="calendar"> block Requirements: group today first, include day/date/time/name/duration, exclude all-day and OOO blocks, output raw HTML to stdout.
Out of v1
Handwritten notes Source hyperlinks per item Cross-source deduplication Reply-to-update Auto-scheduled send Time estimate learning
NavIQ website Scoped High

Agent-assisted workflow from "I want a NavIQ page" to a stakeholder-ready HTML prototype in 2 weeks — no marketing agency required. Covers platform discovery and access setup, page architecture and copy scaffolding using existing NavIQ assets and the /akura-design skill, and a built HTML prototype of the new NavIQ product page plus updated akuramedical.com homepage section. Inputs: logo, animations, spec sheets, in-progress copy. Output: shareable prototype URL + stakeholder feedback template.

Sub-builds
1 Platform discovery + access brief 2–3 hrs
Generate an IT questionnaire (platform, CMS, hosting, credentials owner) and a NavIQ asset inventory. Done when: 5 IT questions answered and asset inventory checklist completed.
2 Page architecture + copy scaffold 4–6 hrs
Define section structure, messaging hierarchy, and CTAs for the NavIQ page and main page update using /akura-design skill. Produce a content outline for stakeholder copy review. Done when: 3-section NavIQ outline with placeholder copy is ready for Marketing, R&D, Reg, and Legal review.
3 HTML prototype 6–8 hrs
Build static HTML/CSS prototype of NavIQ product page and main page update using /akura-design skill, existing animations, and logo. Done when: prototype loads at a shareable Vercel URL.
Starting prompt — Sub-build 1
I am building a new NavIQ product page for akuramedical.com to replace my marketing agency. Before I can design or build anything, I need to answer two questions: what platform is the website built on, and who has backend access. Please do two things: 1. Draft a plain-language IT questionnaire (5–7 questions) I can send to our IT team to identify the website platform, CMS, hosting provider, and who holds the credentials. 2. Create a NavIQ Asset Inventory checklist with the following pre-filled: - NavIQ animations: available - NavIQ logo: available - NavIQ spec sheets: available - Product copy: in development (not final) - Photography: not available Add blank line items for: video demos, clinical outcome data, testimonials/case studies, approved taglines. Note: brand colors and fonts are in the /akura-design skill. Format both as a clean document I can share with the team.
